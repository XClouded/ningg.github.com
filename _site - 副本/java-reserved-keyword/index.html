final

常量命名

示例代码，参考：

public static final String SPOOL_DIRECTORY = "spoolDir";

思考：使用final定义的常量，能够被修改吗？由于是static的变量，因此不需要实例化类的对象，直接调用class就能获取。

示例代码，参考：

private final long length;

思考：上述使用final来标识成员length为常量，但为赋值，是否是在class实例化时，通过构造方法对length进行初始化赋值，并且由于使用final修饰，length在赋值之后不能再被修改？即：

  • final修饰的成员变量，一旦赋值，不能更改；

synchronized

子类override method时，添加synchronized

@Override
public synchronized void configure(Context context) {
	spoolDirectory = context.getString(SPOOL_DIRECTORY);
}
